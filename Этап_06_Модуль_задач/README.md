## Заметки
- Модуль задач построен по образцу модуля мероприятий (Этап 4): server actions
  `tasks/actions.ts` (createTask / updateTask / updateTaskStatus / deleteTask), общая форма
  `TaskForm.tsx`, карточка `tasks/[id]`, редактирование `tasks/[id]/edit`, удаление с
  подтверждением `DeleteTaskButton.tsx`.
- Смена статуса доступна **и в карточке, и прямо в списке** (мини-выпадашка `TaskStatusSelect.tsx`,
  выбор Андрея) — список фильтрованных задач сохраняется после смены через `redirectTo`.
- `/tasks` — фильтры по статусу (чипы) и мероприятию (выпадашка, GET-форма); строки кликабельны.
- Из карточки мероприятия: «+ Добавить задачу» (`/tasks/new?event=<id>`, привязка предзаполнена) и
  ссылки на карточки задач.
- Заметка на Этап 10: `event_id` из формы не сверяется со списком своих мероприятий (низкий риск,
  RLS защищает по `user_id`).
