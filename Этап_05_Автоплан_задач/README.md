# Этап 5. Автоматический план подготовки

## Цель
При создании мероприятия система сама создаёт список задач по дате проведения
(срок задачи = дата мероприятия − offset_days из шаблона).

## Подход — каркас (важно)
Делаем **механику**, а не финальное наполнение:
- Берём **универсальные базовые принципы** организации мероприятия (концепция → площадка →
  бюджет → программа → подрядчики → анонс → подготовка → проведение → отчёт).
  **Без привязки к методичке ФЦ БАС.**
- Пока **один тип** мероприятия — «Познавательное мероприятие». Другие типы добавляются в
  `EVENT_TYPES` (`app/src/lib/events.ts`), под них наполняется `checklist_templates`.
- Реальное наполнение задач, шаблоны документов и доводку Андрей выполнит **на этапе 10**
  («тестирование на себе»), опираясь на свой рабочий опыт.

## Что сделано
- Таблица `checklist_templates` наполнена универсальным набором (~10 задач, тип `educational`);
  добавлена колонка `priority`. Миграция: `app/supabase/migrations/0005_checklist_templates_seed.sql`.
- В форму мероприятия добавлен **выбор типа**; **дата начала стала обязательной**
  (`app/src/app/(cabinet)/events/EventForm.tsx`).
- При создании мероприятия задачи генерируются автоматически — `createTasksForEvent` в
  `app/src/app/(cabinet)/events/actions.ts` (срок = дата − offset_days, приоритет из шаблона).
- Задачи показываются в карточке мероприятия (`events/[id]/page.tsx`) и на странице `/tasks`
  (с привязкой к мероприятию). Хелперы статусов/приоритетов/срока — `app/src/lib/tasks.ts`.

## Чек-лист этапа
- [x] Таблица `checklist_templates` наполнена (универсальный каркас, один тип).
- [x] При создании мероприятия задачи создаются автоматически.
- [x] У каждой задачи правильно посчитан срок (`due_date = дата мероприятия − offset_days`).
- [x] Созданные задачи видны в карточке мероприятия и в `/tasks`.
- [x] Добавлен выбор «Тип мероприятия» в форму; `event_type` пишется в БД.
- [x] Дата начала сделана обязательной.
- [x] Карточка мероприятия: блок «Задачи» — реальный список; исправлена нумерация заглушек
  (Документы — этап 7, Заметки опыта — этап 8).

## Границы (не здесь)
- Смена статуса задачи, отметка «выполнено», ручное создание/редактирование/удаление,
  карточка задачи — **этап 6**.
- Пересчёт сроков при изменении даты мероприятия — не делаем (автоплан разовый, при создании).

## Результат этапа
Пользователь создаёт мероприятие (тип + дата) → система сама создаёт план задач с корректными сроками.
