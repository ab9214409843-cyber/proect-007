Контекст проекта для Claude Code. Читается автоматически в начале каждой сессии в этой папке.

## О проекте

**Proect_007** — веб-приложение (SaaS) для организаторов спортивных мероприятий.
Автор и заказчик — Андрей Башкатов (АНО «ФЦ БАС»). Андрей **не разработчик**: код пишет в основном
Claude, Андрей принимает решения, тестирует «удобно/неудобно» и заводит аккаунты/доступы.

Что приложение должно уметь (MVP):
- создать мероприятие;
- автоматически получить план подготовки (задачи по дате мероприятия);
- вести задачи;
- хранить документы;
- фиксировать опыт после мероприятия.

Полная дорожная карта — в [docs/roadmap.md](docs/roadmap.md). Источник истины по объёму MVP.

## Структура папок

```

├── CLAUDE.md         ← этот файл, главный контекст и прогресс
├── README.md         ← короткая шапка «с чего начать»
```

Важно: **код приложения живёт ТОЛЬКО в `app/`** одной общей структурой. Папки `Этап_XX` — это
документация и чек-листы, в них код не кладём.

## Как мы работаем

- **Новое окно на каждую задачу.** Открываешь папку нужного этапа → читаешь её `README.md` →
  выполняешь чек-лист → отмечаешь статус.
- Claude в любом окне внутри `Proect_007` автоматически видит этот `CLAUDE.md` → контекст ясен сразу.
- Общий прогресс отмечаем здесь, в разделе «Текущий этап» (ниже).
- **По завершению каждого этапа — сразу коммит всех изменений и push в `main`** (Vercel
  автоматически обновит сайт). Отдельного разрешения спрашивать не нужно.
- Стиль общения с Андреем: **русский, на «ты», дружески, кратко и по делу, без технического жаргона.**
  Объясняем без терминов, даём конкретные пошаговые инструкции.
