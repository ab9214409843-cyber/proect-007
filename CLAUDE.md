Контекст проекта для Claude Code. Читается автоматически в начале каждой сессии в этой папке.

## О проекте

**Proect_007** — веб-приложение (SaaS) для организаторов спортивных мероприятий.
Автор и заказчик — Андрей Башкатов (АНО «ФЦ БАС»). Андрей **не разработчик**: код пишет в основном
Claude, Андрей принимает решения, тестирует «удобно/неудобно» и заводит аккаунты/доступы.

Что приложение должно уметь (MVP):
- создать мероприятие;
- автоматически получить план подготовки (задачи по дате мероприятия);
- вести задачи;
- хранить документы;
- фиксировать опыт после мероприятия.

Полная дорожная карта — в [docs/roadmap.md](docs/roadmap.md). Источник истины по объёму MVP.

### Технические заметки (важно для будущих окон)
- **Next.js 16, не 15.** Есть breaking changes (см. `app/AGENTS.md`): middleware → **`proxy.ts`**
  (функция `proxy`, runtime Node.js); `cookies()`, `params`, `searchParams` — **асинхронные** (через `await`);
  Turbopack по умолчанию; `next lint` убран. Перед правкой Next-кода сверяйся с `app/node_modules/next/dist/docs/`.
- **Supabase подключён через `@supabase/ssr`** (актуальная библиотека, не устаревшие auth-helpers):
  `app/src/lib/supabase/{client,server,session}.ts`, защита маршрутов — `app/src/proxy.ts`.
- **Секреты:** ключи Supabase в `app/.env.local` (под `.gitignore`, в git НЕ попадают).
  Access token Supabase — в `C:\Users\ab921\.claude.json` (вне проекта).
- **Деплой на Vercel:** при подключении указать **Root Directory = `app`** (приложение вложено в подпапку).
- Node/npm/git ставились внутри открытого VSCode → после установки нужен был перезапуск VSCode (PATH).
- MCP-серверы на Windows запускаются через `cmd /c npx ...`.

## Приоритеты MVP / что НЕ делаем

**Обязательно для MVP:** авторизация, главная страница, личный кабинет, мероприятия, автосоздание
задач, задачи, документы, база опыта.

**НЕ делаем в MVP:** мобильное приложение, билеты, регистрацию участников, CRM, маркетинг, сложную
аналитику, интеграции, оплату подписки, AI-генерацию документов. (Эти пункты — в Этап 11 / версию 1.1.)
