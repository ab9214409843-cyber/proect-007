# Этап 9. Минимальный дизайн

## Цель
Привести интерфейс к единому чистому SaaS-виду: боковое меню, шапка, карточки, таблицы, формы, адаптивность.

## Задача для Claude (новое окно)
> Приведи интерфейс к единому SaaS-дизайну: sidebar, header, карточки, таблицы, формы, адаптивность.

## Стиль
- чистый SaaS-интерфейс;
- светлая тема;
- боковое меню;
- карточки и таблицы;
- минимум визуального шума.

## Чек-лист этапа
- [x] Sidebar (боковое меню по разделам кабинета).
- [x] Header (шапка).
- [x] Cards (карточки).
- [x] Tables (таблицы) — списки-строки в едином стиле (`rowCard`).
- [x] Modals (модальные окна) — `Modal` + `ConfirmDeleteButton` вместо браузерного `confirm`.
- [x] Forms (единый стиль форм).
- [x] Empty states (пустые состояния — «пока ничего нет»).
- [x] Адаптивность (мобильное меню-гамбургер, нормально смотрится на узком экране).

## Результат этапа
Все экраны выглядят единообразно и аккуратно, есть общий каркас (sidebar + header) и проработанные пустые состояния.

## Статус
`завершён ✅`

## Заметки
Решения Андрея: гамма — строгая чёрно-белая (монохром), подтверждение удаления — своё модальное окно.

Что сделано (унификация + лоск, без смены общего вида):
- **Общий UI-набор** `app/src/components/`: строки-классы `ui.ts` (кнопки/поля/карточки/бейджи/чипы),
  компоненты `PageHeader`, `EmptyState`, `Modal`, `ConfirmDeleteButton`. Это **единый источник стиля** —
  дальше менять вид приложения = править эти файлы.
- **Модальное окно удаления** заменило браузерный `confirm` во всех 4 модулях (events/tasks/documents/
  experience); 4 одинаковых файла `Delete*Button.tsx` удалены.
- **Мобильное меню**: пункты вынесены в `nav-items.ts`; добавлен `MobileNav.tsx` (гамбургер + выпадающая
  панель) для узких экранов, сайдбар остаётся на ≥sm.
- **Починены огрехи** `globals.css`: подключён шрифт Geist (был перебит Arial), удалена авто-тёмная тема
  (продукт светлый).
- Набор применён по всем экранам: списки, карточки, формы, авторизация, дашборд. Минимальная длина
  пароля в форме входа выровнена до 8.
- Проверено: `npm run build` чистый; Playwright — главная/вход/дашборд/карточка, модалка удаления
  (открытие, «Отмена», вид), мобильное меню на 375px; ошибок в консоли нет. Тестовые данные удалены.

## Дизайн v2 — «тёплая редакционная роскошь» (2026-06-10) ✅

По итогам показа Андрей решил, что чёрно-белый вид «слишком прост». Сделан полноценный визуальный
апгрейд по образцу **manifestos.com/compliance**. Решения Андрея: **палитра крем + терракота**,
заголовки **Fraunces**, **тёплые фото** на главной. Продукт переименован (видимо) в **EventOS**.

Благодаря набору выше перекраска свелась к смене **токенов** в `globals.css` (именованные цвета
Tailwind v4: `paper`/`paper-2`/`card`/`espresso`/`muted`/`sand`/`clay`) + правке строк в `ui.ts`,
плюс добавлен засечный шрифт `Fraunces` (`layout.tsx`, utility `font-serif`). Что сделано:
- **Палитра**: кремовый фон, кофейный текст/кнопки, терракотовый акцент (ссылки, надзаголовки,
  активные пункты, фокус полей). Кнопки/чипы стали «пилюлями».
- **Типографика**: заголовки и вордмарк — Fraunces; `PageHeader` получил серифный заголовок и
  необязательный надзаголовок-`eyebrow` (капс вразрядку, цвет `clay`).
- **Главная** (`page.tsx`) переписана в редакционном стиле: полоса-анонс, hero с надзаголовком и
  крупным засечным заголовком, два тёплых фото (`public/images/hero-hall.jpg`, `banquet.jpg`,
  Unsplash, free license), чередующиеся секции, espresso-футер.
- **Кабинет/формы/вход/служебные экраны** перекрашены сметанием «серых» утилит на тёплые токены;
  заголовки карточек — серифные.
- Переименование в **EventOS**: вордмарк (кабинет + главная), `metadata.title`. Технические
  идентификаторы (репозиторий, проект Supabase) не трогали.
- Проверено: `npm run build` чистый; Playwright — главная (desktop+375px), вход, дашборд, форма
  (терракотовый фокус), карточка + модалка, гамбургер на 375px; консоль без ошибок; тестовый аккаунт
  создан и удалён.

> Примечание (Windows): после массовой замены через `sed` dev-сервер мог не пересобрать часть
> маршрутов (file-watcher пропускает sed-записи) — лечится перезапуском `npm run dev` с чисткой `.next`.
