## Цель
Привести интерфейс к единому чистому SaaS-виду: боковое меню, шапка, карточки, таблицы, формы, адаптивность.

## Задача для Claude (новое окно)
> Приведи интерфейс к единому SaaS-дизайну: sidebar, header, карточки, таблицы, формы, адаптивность.

## Стиль
- чистый SaaS-интерфейс;
- светлая тема;
- боковое меню;
- карточки и таблицы;
- минимум визуального шума.

`завершён ✅`

## Заметки
Решения Андрея: гамма — строгая чёрно-белая (монохром), подтверждение удаления — своё модальное окно.

Что сделано (унификация + лоск, без смены общего вида):
- **Общий UI-набор** `app/src/components/`: строки-классы `ui.ts` (кнопки/поля/карточки/бейджи/чипы),
  компоненты `PageHeader`, `EmptyState`, `Modal`, `ConfirmDeleteButton`. Это **единый источник стиля** —
  дальше менять вид приложения = править эти файлы.
- **Модальное окно удаления** заменило браузерный `confirm` во всех 4 модулях (events/tasks/documents/
  experience); 4 одинаковых файла `Delete*Button.tsx` удалены.
- **Мобильное меню**: пункты вынесены в `nav-items.ts`; добавлен `MobileNav.tsx` (гамбургер + выпадающая
  панель) для узких экранов, сайдбар остаётся на ≥sm.
- **Починены огрехи** `globals.css`: подключён шрифт Geist (был перебит Arial), удалена авто-тёмная тема
  (продукт светлый).
- Набор применён по всем экранам: списки, карточки, формы, авторизация, дашборд. Минимальная длина
  пароля в форме входа выровнена до 8.
- Проверено: `npm run build` чистый; Playwright — главная/вход/дашборд/карточка, модалка удаления
