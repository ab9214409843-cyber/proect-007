# Этап 4. Модуль мероприятий

## Цель
Сделать полный CRUD по мероприятиям: список, создание, карточка, редактирование, удаление, смена статуса.

## Задача для Claude (новое окно)
> Сделай CRUD для мероприятий: создание, список, редактирование, удаление, карточка мероприятия.

## Чек-лист этапа

Список мероприятий (`/events`):
- [x] Посмотреть все мероприятия.
- [x] Создать мероприятие.
- [x] Открыть карточку мероприятия.
- [x] Изменить статус мероприятия.

Карточка мероприятия (`/events/[id]`) показывает:
- [x] название;
- [x] дату;
- [x] место;
- [x] статус;
- [~] задачи (привязанные) — блок-заглушка, наполнится на этапе 5;
- [~] документы (привязанные) — блок-заглушка, наполнится на этапе 6;
- [~] заметки опыта (привязанные) — блок-заглушка, наполнится на этапе 7.

Статусы мероприятия:
- планируется
- в подготовке
- проводится
- завершено

## Результат этапа
Мероприятия можно создавать, смотреть списком, открывать карточку, менять статус, редактировать и удалять.

## Статус
`завершён ✅`

## Заметки
- Код: `app/src/app/(cabinet)/events/` (список, `new`, `[id]`, `[id]/edit`, `actions.ts`,
  `EventForm.tsx`, `DeleteEventButton.tsx`) + общие хелперы `app/src/lib/events.ts`.
- Логика изменений — server actions (создание/правка/смена статуса/удаление); удаление с подтверждением.
- Проверено в браузере (Playwright) end-to-end + подтверждено в Supabase; новых миграций не потребовалось.
- Привязанные задачи/документы/опыт на карточке — пока блоки-заглушки (модули появятся на этапах 5–8).

## Найдено при комплексной проверке 2026-06-10 (исправим на следующих этапах)
- ⚠️ **В форме нет поля «Тип мероприятия»** → `events.event_type` остаётся пустым. Это нужно автоплану
  (этап 5) для выбора шаблона задач. Добавить выбор типа — задача [Этапа 5](../Этап_05_Автоплан_задач/README.md).
- **Нумерация этапов в заглушках карточки** `[id]/page.tsx` неточная: «Документы — этап 6,
  Заметки опыта — этап 7». По дорожной карте документы = **этап 7**, опыт = **этап 8**. Исправить при
  доработке карточки (этап 5) или на финальной вычитке (этап 10).
- **`updateEventStatus` и `deleteEvent` не проверяют ошибку записи** (молча игнорируют) — подчистить
  на [этапе 10](../Этап_10_Тестирование/README.md). Данные защищены RLS, так что сейчас не критично.
