# Этап 8. Блок опыта

## Цель
После мероприятия пользователь фиксирует выводы по категориям — копится база опыта.

## Задача для Claude (новое окно)
> Сделай блок опыта: заметки после мероприятия, категории, связь с мероприятием.

## Что фиксируем
- что прошло хорошо;
- какие были ошибки;
- что улучшить в следующий раз;
- какие подрядчики сработали хорошо/плохо;
- важные выводы.

## Чек-лист этапа

Раздел `/experience`:
- [x] Создать заметку опыта.
- [x] Указать категорию.
- [x] Привязать к мероприятию.
- [x] Редактировать/удалять заметку.
- [x] Заметки видны в карточке мероприятия.

Категории:
- организация, площадка, подрядчики, документы, участники, бюджет, другое.

## Результат этапа
После мероприятия можно записывать выводы по категориям; они связаны с мероприятием и копятся в базе опыта.

## Статус
`завершён ✅`

## Заметки
- Полный модуль по образцу задач/документов: список с фильтрами `/experience`, создание
  `/experience/new`, карточка `/experience/[id]`, редактирование `/experience/[id]/edit`,
  удаление с подтверждением. Логика — server actions `experience/actions.ts`; форма
  `ExperienceForm.tsx`; типы/категории/бейджи — `app/src/lib/experience.ts`.
- Добавлено второе измерение **«тип вывода»** (✅ что хорошо / ⚠️ ошибка / 💡 что улучшить) —
  отдельно от темы (категории). Под него — миграция
  `app/supabase/migrations/0007_experience_notes_kind.sql` (колонка `kind`).
- Фильтры на списке: по типу вывода (чипы), категории и мероприятию (выпадашки).
- В карточке мероприятия — секция «Заметки опыта» + «+ Добавить заметку» (с предзаполнением
  мероприятия).
- Проверено: `npm run build` чистый, `get_advisors` без новых замечаний, ручной прогон CRUD
  в браузере (создание/фильтры/редактирование/удаление/привязка к мероприятию), тестовые данные
  удалены.
