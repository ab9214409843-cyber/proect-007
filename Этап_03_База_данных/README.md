# Этап 3. База данных

## Цель
Создать в Supabase основные таблицы и настроить доступ так, чтобы данные можно было создавать,
читать, редактировать и удалять (CRUD).

## Задача для Claude (новое окно)
> Создай структуру базы данных для мероприятий, задач, документов, шаблонов чек-листов и заметок
> опыта. Подготовь SQL для Supabase. Настрой RLS (каждый пользователь видит только свои записи).

## Таблицы (подробности — в [../docs/db-schema.md](../docs/db-schema.md))
- [ ] `users` — профиль пользователя
- [ ] `events` — мероприятия
- [ ] `tasks` — задачи
- [ ] `documents` — документы
- [ ] `checklist_templates` — шаблоны задач для автоплана
- [ ] `experience_notes` — заметки опыта

## Чек-лист этапа
- [ ] Написан SQL для всех 6 таблиц.
- [ ] SQL применён в Supabase (таблицы созданы).
- [ ] Включён RLS, настроены политики «вижу только своё».
- [ ] Проверено: запись можно создать, прочитать, изменить, удалить.

## Результат этапа
База готова; данные можно создавать, читать, редактировать и удалять.
