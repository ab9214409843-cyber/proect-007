# Этап 3. База данных

## Цель
Создать в Supabase основные таблицы и настроить доступ так, чтобы данные можно было создавать,
читать, редактировать и удалять (CRUD).

## Задача для Claude (новое окно)
> Создай структуру базы данных для мероприятий, задач, документов, шаблонов чек-листов и заметок
> опыта. Подготовь SQL для Supabase. Настрой RLS (каждый пользователь видит только свои записи).

## Заметки
- **6 таблиц** созданы в Supabase 4 миграциями (через MCP). Копии SQL — в репозитории:
  [app/supabase/migrations/](../app/supabase/migrations/) (`0001_core_schema`, `0002_profile_trigger`,
  `0003_rls_policies`, `0004_lockdown_trigger_function`).
- **Статусы/типы/категории — английскими кодами** с проверкой (CHECK), русские подписи будут в
  интерфейсе на этапах 4–8. Расшифровка кодов — в комментариях к колонкам в БД и в `0001_core_schema.sql`.
  Например: статус мероприятия `planned / preparing / in_progress / done`; статус задачи
  `todo / in_progress / review / done`; приоритет `low / medium / high`.
- **`event_id` у задач/документов/заметок — необязательный** (задача может быть без мероприятия,
  см. Этап 6). При удалении мероприятия связанные задачи/документы/заметки удаляются каскадом.
- **Профиль `users` заполняется автоматически** триггером `on_auth_user_created` при регистрации
  (берёт `id` и `email` из авторизации; имя пока пустое — при регистрации не спрашивается).
  Существующий тестовый аккаунт добавлен в профиль через backfill.
- **RLS включён на всех 6 таблицах**: владелец видит/меняет только свои записи (`auth.uid() = user_id`);
  `checklist_templates` — справочник, только чтение для вошедших.
- **Проверено:** структура и связи (`list_tables`), отсутствие проблем RLS (`get_advisors security` —
  чисто), CRUD-смоук в SQL (создать → прочитать → обновить → проверка CHECK → каскадное удаление),
  backfill профиля, `npm run build` без ошибок (новый `app/src/lib/supabase/database.types.ts`
  компилируется).
- **Полную проверку «вижу только своё»** (два пользователя) делаем на Этапе 4, когда появятся формы
  создания мероприятий в браузере.
- **Сгенерированы TypeScript-типы** базы → [app/src/lib/supabase/database.types.ts](../app/src/lib/supabase/database.types.ts)
  (перегенерировать после изменения схемы). Будут использоваться кодом этапов 4–8.
- ⚠️ К публичному запуску (вместе с возвратом подтверждения email): включить в Supabase Auth защиту от
  утёкших паролей (Leaked Password Protection) — сейчас отключена.
