# Стек и окружение

## Что используем и зачем (простыми словами)

| Технология | Что делает | Почему выбрали |
|---|---|---|
| **Next.js** | Каркас сайта/приложения (страницы, переходы, серверная часть) | Стандарт для современных веб-приложений, всё в одном |
| **TypeScript** | Язык кода с проверкой ошибок | Меньше ошибок, код понятнее |
| **Tailwind CSS** | Оформление интерфейса (цвета, отступы, кнопки) | Быстро собирать аккуратный вид без отдельных файлов стилей |
| **Supabase** | База данных + вход по email/паролю + хранение файлов | Три вещи в одном сервисе, бесплатный тариф на старте |
| **Vercel** | Публикация приложения в интернете | Бесплатно, связывается с GitHub в пару кликов |
| **GitHub** | Хранилище кода | Уже есть; через него Vercel публикует сайт |

Всё перечисленное на старте — **бесплатно**. Платить понадобится только при росте нагрузки/объёма.

## Чек-лист подготовки окружения (перед написанием кода)

1. **Node.js LTS** — движок, без которого приложение не запустить.
   - Установка: `winget install OpenJS.NodeJS.LTS`
   - Проверка: `node -v` и `npm -v` должны показать версии.
2. **Git** — для хранения кода и связки с GitHub.
   - Установка: `winget install Git.Git`
   - Проверка: `git -v`.
3. **Supabase** — регистрация на supabase.com, создать новый проект (запомнить пароль БД и ключи).
4. **Vercel** — регистрация на vercel.com через GitHub.
5. **MCP-расширения** — подключаем, когда есть Supabase-проект:
   - Supabase MCP — чтобы Claude управлял базой прямо из этого окна.
   - Браузерный MCP — чтобы Claude визуально проверял и тестировал интерфейс.
