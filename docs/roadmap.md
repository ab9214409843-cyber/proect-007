# Дорожная карта разработки MVP

> Источник истины по объёму MVP. Исходный документ Андрея, сохранён без изменений.

## Цель MVP

Создать веб-приложение для организатора мероприятий, где можно:

* создать мероприятие;
* автоматически получить план подготовки;
* вести задачи;
* хранить документы;
* фиксировать опыт после мероприятия.

Первая версия делается под организаторов спортивных мероприятий.

---

# ЭТАП 1. Подготовка проекта

## Задача 1. Зафиксировать стек

Рекомендуемый стек:

* Frontend: Next.js
* Язык: TypeScript
* UI: Tailwind CSS
* База данных: Supabase
* Авторизация: Supabase Auth
* Хранение файлов: Supabase Storage
* Деплой: Vercel

Результат этапа:

* создан проект;
* настроена структура папок;
* подключен Supabase;
* приложение запускается локально.

# ЭТАП 5. Автоматический план подготовки

## Задача 7. Генерация задач по дате мероприятия

При создании мероприятия пользователь указывает дату.
Система автоматически создает задачи по шаблону.

Пример:

За 90 дней:
* определить концепцию;
* выбрать площадку.

За 60 дней:
* подготовить положение;
* согласовать подрядчиков.

За 30 дней:
* подготовить сценарий;
* заказать наградную продукцию.

За 7 дней:
* проверить логистику;
* провести финальную сверку.

За 1 день:
* проверить площадку;
* подтвердить готовность команды.

Результат этапа:

* пользователь создает мероприятие;
* система сама создает план задач.

# Приоритет разработки

## Обязательно для MVP

1. Авторизация
2. Главная страница
3. Личный кабинет
4. Мероприятия
5. Автоматическое создание задач
6. Задачи
7. Документы
8. База опыта

## Не делать в MVP

* мобильное приложение;
* билеты;
* регистрацию участников;
* CRM посетителей;
* маркетинг;
* сложную аналитику;
* интеграции;
* оплату подписки;
* AI-генерацию документов.
